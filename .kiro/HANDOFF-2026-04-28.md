### Sources (16 RSS feeds)
Added in `lib/content-sources.ts`. 4 new sources added **today**: KTAR Phoenix, 12 News Phoenix, AZFamily (3TV/CBS5), AZ Big Media. Also: Mesa Nehemiah Zoning newsletter, NOAA weather, r/phoenix, r/arizona, historic-property RSS, AZ state feeds.

### Flow

```
RSS/HTML feeds → content-ingest Lambda → S3 data lake + DDB index
                                         ↓
                              content-bundler Lambda (clusters by topic/time)
                                         ↓
                              content-drafter Lambda (Claude Haiku 4.5 + Bedrock)
                                         ↓
                              photo-finder Lambda (5-tier photo cascade)
                                         ↓
                              CONTENT#DRAFT#{draftId} in DDB → dashboard review UI
                                         ↓
                              owner clicks Approve → CONTENT#BLOG#{slug} + triggers GHA rebuild → live
```

### Drafter
- Model: `us.anthropic.claude-haiku-4-5-20251001-v1:0` (switched from Nova Micro for better structured output)
- Max tokens: 3500
- Bundle cap: 20 items (was 8) — Claude has 200K context
- Prompt rewritten for **opinionated local Realtor voice**: no rhetorical questions in "What to do next", must have real CTAs, citation rule tightened
- `stripTrailingDisclaimer()` prevents duplicate disclaimers

### Dedup (3 layers)
- **Bundler**: `readCooldownTopics()` queries `BLOG#PUBLISHED` GSI, drops any bundle whose topic was published <7 days ago
- **Drafter**: `readRecentPublished()` passes last 50 published {title, slug, topic, date} as "DO NOT DUPLICATE" block in the prompt
- **Approve handler**: refuses to overwrite existing `CONTENT#BLOG#{slug}`, throws `VALIDATION_ERROR`

### Photo finder — 5-tier cascade
1. **Google Street View** — for bundles containing street addresses (regex extract). Metadata check first to avoid billing for "no imagery" hits. Uses existing `mesahomes/live/google-maps-api-key`.
2. **Wikimedia** — Arizona-locality filtered
3. **Pexels** — `mesahomes/live/pexels-api-key` **NOT YET CREATED BY OWNER**
4. **Unsplash** — already had
5. **Bedrock Titan Image Generator v2** — AI Mesa scene, $0.008/image, negative prompt for text/watermark/logo
6. **CURATED_FALLBACKS** — currently just logo emblem; owner should upload real Mesa photos

All tiers filter via `isLocalToArizona()` and log raw-vs-filtered counts.

### Article file pattern
Each article has:
1. Metadata export (title/description/canonical/openGraph)
2. `articleJsonLd` const (`articleSection: 'Selling Guides'`)
3. `faqJsonLd` const with 5+ FAQs
4. Default export: Header → FullServiceUpgradeBanner → main (both JSON-LDs) → ArticleBody → Footer → StickyContactBar
5. ArticleBody: breadcrumb → H1 → lede → `<div className="prose">{<ArticleSections />}`
6. H2 sections: `mt-8 mb-3 font-heading text-xl font-bold text-charcoal`
7. Close with border-primary bg-warm-beige/40 CTA block
8. `<hr>` then disclaimer `<p className="text-xs text-text-light italic">`

### Voice constraints (hard-enforced)
- No em-dashes
- No "whether you're X or Y" pivots
- No hollow words: leverage, ecosystem, game-changer, navigate, ensure, utilize, streamline, enhance
- No "that said," "in today's market," "ever-evolving," "dynamic," "vibrant," "thriving"
- No tripartite adjective lists ("modern, sleek, stylish")
- Short sentences, contractions OK, opinionated stance required
- Specific numbers (ZIPs, dollar amounts, days on market)
- Apostrophes in JSX escaped as `&apos;`

## 9. Data Layer Access Patterns

DynamoDB single-table design. Everything in `mesahomes-main` table:

```
# Ingest
PK = SOURCE#{sourceId}       SK = HASH#{hash}                          # dedup
GSI1PK = TOPIC#{topic}        GSI1SK = SEEN#{date}#{src}#{itemId}       # bundler reads

# Bundles
PK = BUNDLE#{topic}#{date}    SK = v{n}#{bundleId}
GSI1PK = BUNDLE#PENDING                                                 # drafter reads

# Drafts
PK = CONTENT#DRAFT#{draftId}  SK = v1
GSI1PK = DRAFT#PENDING|DRAFT#PUBLISHED|DRAFT#REJECTED                   # review flow

# Published content
PK = CONTENT#BLOG#{slug}      SK = v1
GSI1PK = BLOG#PUBLISHED                                                  # blog feed
```
